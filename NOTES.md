# Git課題 メモ

## 課題1

### git initの前後でディレクトリの中身に何が増えたか
.git という隠しフォルダが作られた。
Gitの管理情報（履歴・設定など）がすべてここに入っている。

### ステージとは何か
コミットする予定のファイルを一時的に置く場所。
git add でファイルをステージに上げる。

### コミットとは何か
ステージの内容をスナップショットとしてGitの歴史に記録すること。

## 課題2

### "+"と"-"はそれぞれ何を表しているか
+ は追加された行、- は削除された行を表す。

### コミットした後にもう一度差分を表示すると何が表示されるか
何も表示されない。
コミット後はワーキングツリーとリポジトリの内容が一致するため、差分がゼロになるから。

## 課題14

### 退避した変更はgit logには表示されない。どうやって一覧や中身を確認できるか
git stash list   → 退避した一覧を確認
git stash show -p → 退避した中身を確認

### git stash popとgit stash applyの違いは何か
pop:   退避を復元してstashから削除する
apply: 退避を復元してもstashに残り続ける

## 課題15

### コンフリクトが起きたとき、ファイルにはどのような印が入るか
<<<<<<< HEAD    → 現在のブランチの内容
=======         → 境界線
>>>>>>> branch-B → マージしようとしたブランチの内容

### 最終的な内容をどう決めたか
両方の変更を反映させて「version A & version B」とした。
